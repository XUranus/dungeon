# Dashboard API

公共大屏相关接口，**无需认证**即可访问。公共聊天接口基于 IP 地址进行每日频率限制。

**路由前缀：** `/api/dashboard`

---

## GET /api/dashboard/summary

获取最新的观点摘要列表及剩余聊天次数。

### 请求

```
GET /api/dashboard/summary?limit=20
```

**查询参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `limit` | `integer` | `20` | 返回条数上限，最大 50 |

### 响应

**成功 (200)：**

```json
{
  "items": [
    {
      "id": 42,
      "platform": "zhihu",
      "title": "如何看待当前A股市场",
      "content_preview": "我认为当前市场处于结构性行情中，科技板块依然值得关注...",
      "content_type": "answer",
      "url": "https://www.zhihu.com/question/123/answer/456",
      "like_count": 128,
      "published_at": "2024-06-15T10:30:00"
    },
    {
      "id": 41,
      "platform": "zsxq",
      "title": null,
      "content_preview": "今天的操作思路：逢低布局港股科技ETF...",
      "content_type": "talk",
      "url": "https://wx.zsxq.com/topic/789",
      "like_count": 35,
      "published_at": "2024-06-14T08:00:00"
    }
  ],
  "chat_remaining": 10
}
```

**响应字段说明：**

| 字段 | 说明 |
|------|------|
| `items` | 摘要列表，按发布时间倒序 |
| `items[].content_preview` | 内容前 200 字符预览，超出部分以 `...` 截断 |
| `items[].content_type` | 内容类型：`answer`、`article`、`pin`、`talk`、`q&a` |
| `chat_remaining` | 当前默认每日免费问答次数上限 |

### curl 示例

```bash
curl "http://localhost:8000/api/dashboard/summary?limit=10"
```

## GET /api/dashboard/chat-remaining

查询当前客户端 IP 地址的每日聊天使用情况。

### 请求

```
GET /api/dashboard/chat-remaining
```

### 响应

**成功 (200)：**

```json
{
  "used": 3,
  "limit": 10,
  "remaining": 7
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `used` | `integer` | 今日已使用次数 |
| `limit` | `integer` | 每日总次数上限（配置项 `public_chat_daily_limit`） |
| `remaining` | `integer` | 今日剩余次数（`max(0, limit - used)`） |

:::note
频率限制基于客户端 IP 地址，存储在内存中（非持久化），服务重启后计数归零。
:::

### curl 示例

```bash
curl http://localhost:8000/api/dashboard/chat-remaining
```

## POST /api/dashboard/chat

公共 RAG 问答接口，基于 SSE 流式返回 LLM 回答。**无需认证**，但受 IP 级每日频率限制。

### 请求

```
POST /api/dashboard/chat
Content-Type: application/json
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | `string` | 是 | 用户提问内容 |
| `history` | `array` | 否 | 多轮对话历史（可选） |

**`history` 数组元素结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | `string` | `"user"` 或 `"assistant"` |
| `content` | `string` | 消息内容 |

**请求示例（单轮）：**

```json
{
  "message": "星主对当前美股科技股怎么看？"
}
```

**请求示例（多轮）：**

```json
{
  "message": "那港股呢？",
  "history": [
    {"role": "user", "content": "星主对当前美股科技股怎么看？"},
    {"role": "assistant", "content": "根据星主近期在知识星球的发言..."}
  ]
}
```

### 响应

**成功 (200)：** SSE 流式响应

```
data: 根据
data: 星主
data: 在
data: 知识星球
data: 上的
data: 近期发言
data: ...
data: [DONE]
```

**错误 (429) -- 频率限制：**

```json
{
  "detail": "今日免费问答次数已用完（10次），请明天再试"
}
```

### curl 示例

```bash
curl -N -X POST http://localhost:8000/api/dashboard/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "星主对新能源板块怎么看？"}'
```

:::tip
使用 `-N` 参数禁用 curl 的缓冲，以便实时看到流式输出。
:::

---

## SSE 流式格式详解

`/api/dashboard/chat` 和 `/api/chat` 两个聊天端点均使用相同的 SSE 格式。

### 数据帧格式

每条 SSE 消息由 `data: ` 前缀和实际内容组成，以双换行 `\n\n` 分隔：

```
data: 你好\n\n
data: ，\n\n
data: 根据\n\n
data: [DONE]\n\n
```

### 客户端处理流程

```mermaid
flowchart TD
    A[发送 POST 请求] --> B[解析 SSE 流]
    B --> C{读取一行}
    C -->|data: 内容| D[拼接到回答文本]
    C -->|data: [DONE]| E[流结束]
    C -->|连接断开| E
    D --> C
    E --> F[显示完整回答]
```

### JavaScript 解析示例

```javascript
const response = await fetch('/api/dashboard/chat', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({message: '你的问题'}),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();
let answer = '';

while (true) {
  const {done, value} = await reader.read();
  if (done) break;

  const text = decoder.decode(value);
  const lines = text.split('\n');

  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const content = line.slice(6);
      if (content === '[DONE]') break;
      answer += content;
    }
  }
}
```
