# Dashboard API

The public dashboard endpoints provide data for the visitor-facing display. **No authentication is required.** The public chat endpoint uses cookie-based visitor identification for rate limiting.

**Route prefix:** `/api/dashboard`

---

## GET /api/dashboard/summary

Retrieve the latest opinion summaries and the default daily chat quota.

### Request

```
GET /api/dashboard/summary?limit=20
```

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | `integer` | `20` | Maximum number of items to return (max 50) |

### Response

**Success (200):**

```json
{
  "items": [
    {
      "id": 42,
      "platform": "zhihu",
      "title": "How to view the current A-share market",
      "content_preview": "I believe the current market is in a structural trend, the tech sector remains worth watching...",
      "content_type": "answer",
      "url": "https://www.zhihu.com/question/123/answer/456",
      "like_count": 128,
      "published_at": "2024-06-15T10:30:00"
    },
    {
      "id": 41,
      "platform": "zsxq",
      "title": null,
      "content_preview": "Today's strategy: accumulate Hong Kong tech ETFs on dips...",
      "content_type": "talk",
      "url": "https://wx.zsxq.com/topic/789",
      "like_count": 35,
      "published_at": "2024-06-14T08:00:00"
    }
  ],
  "chat_remaining": 10
}
```

**Response Fields:**

| Field | Description |
|-------|-------------|
| `items` | Summary list, ordered by `published_at` descending |
| `items[].content_preview` | First 200 characters of content, truncated with `...` |
| `items[].content_type` | One of: `answer`, `article`, `pin`, `talk`, `q&a` |
| `chat_remaining` | Default daily free chat quota |

### curl Example

```bash
curl "http://localhost:8000/api/dashboard/summary?limit=10"
```

## POST /api/dashboard/visitor

Register a new visitor or retrieve an existing visitor ID via cookie. The frontend calls this on initial page load to establish a session.

### Request

```
POST /api/dashboard/visitor
Cookie: visitor_id=<existing-id>   (optional, for returning visitors)
```

### Response

**Success (200):**

```json
{
  "visitor_id": "v_a1b2c3d4e5f6"
}
```

The response also sets a `Set-Cookie` header with the `visitor_id`. If the request already contains a valid `visitor_id` cookie, the same ID is returned.

### Response Headers

```
Set-Cookie: visitor_id=v_a1b2c3d4e5f6; Path=/; Max-Age=31536000; SameSite=Lax
```

### curl Example

```bash
# New visitor
curl -X POST http://localhost:8000/api/dashboard/visitor

# Returning visitor
curl -X POST http://localhost:8000/api/dashboard/visitor \
  -b "visitor_id=v_a1b2c3d4e5f6"
```

## GET /api/dashboard/messages

Retrieve the chat message history for a specific visitor.

### Request

```
GET /api/dashboard/messages
Cookie: visitor_id=v_a1b2c3d4e5f6
```

### Response

**Success (200):**

```json
{
  "visitor_id": "v_a1b2c3d4e5f6",
  "messages": [
    {
      "role": "user",
      "content": "What does the host think about the tech sector?"
    },
    {
      "role": "assistant",
      "content": "Based on recent posts, the host is bullish on AI infrastructure..."
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `visitor_id` | `string` | The visitor identifier |
| `messages` | `array` | Chronological message list |
| `messages[].role` | `string` | `"user"` or `"assistant"` |
| `messages[].content` | `string` | Message content |

### curl Example

```bash
curl http://localhost:8000/api/dashboard/messages \
  -b "visitor_id=v_a1b2c3d4e5f6"
```

## POST /api/dashboard/chat

Public RAG question-answering endpoint. Streams the LLM response via SSE. **No authentication required**, but rate-limited per visitor cookie (daily limit).

### Request

```
POST /api/dashboard/chat
Content-Type: application/json
Cookie: visitor_id=v_a1b2c3d4e5f6
```

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | `string` | Yes | The user's question |
| `history` | `array` | No | Multi-turn conversation history |

**`history` Array Element Structure:**

| Field | Type | Description |
|-------|------|-------------|
| `role` | `string` | `"user"` or `"assistant"` |
| `content` | `string` | Message content |

**Request Example (single turn):**

```json
{
  "message": "What does the host think about US tech stocks?"
}
```

**Request Example (multi-turn):**

```json
{
  "message": "What about Hong Kong tech stocks?",
  "history": [
    {"role": "user", "content": "What does the host think about US tech stocks?"},
    {"role": "assistant", "content": "Based on recent posts in the knowledge community..."}
  ]
}
```

### Response

**Success (200):** SSE stream with plain text deltas

```
data: Based
data:  on
data:  the
data:  host's
data:  recent
data:  posts
data: ...
data: [DONE]
```

**Error (429) -- Rate Limit Exceeded:**

```json
{
  "detail": "Daily free chat quota exhausted (10 times), please try again tomorrow"
}
```

### curl Example

```bash
curl -N -X POST http://localhost:8000/api/dashboard/chat \
  -H "Content-Type: application/json" \
  -b "visitor_id=v_a1b2c3d4e5f6" \
  -d '{"message": "What does the host think about the new energy sector?"}'
```

:::tip
Use the `-N` flag to disable curl output buffering, allowing you to see the stream in real time.
:::

## SSE Streaming Format (Dashboard)

The dashboard chat endpoint uses a simple text-delta SSE format.

### Data Frame Format

Each SSE message consists of a `data: ` prefix followed by the content, separated by double newlines (`\n\n`):

```
data: Hello\n\n
data: ,\n\n
data: according\n\n
data: [DONE]\n\n
```

### Client-Side Parsing Flow

```mermaid
flowchart TD
    A[Send POST Request] --> B[Parse SSE Stream]
    B --> C{Read Line}
    C -->|data: content| D[Append to Answer]
    C -->|data: [DONE]| E[Stream Complete]
    C -->|Connection dropped| E
    D --> C
    E --> F[Display Full Answer]
```

### JavaScript Parsing Example

```javascript
const response = await fetch('/api/dashboard/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include',  // send cookies
  body: JSON.stringify({ message: 'Your question here' }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();
let answer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const text = decoder.decode(value);
  const lines = text.split('\n');

  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const content = line.slice(6);
      if (content === '[DONE]') break;
      answer += content;
    }
  }
}

console.log('Full answer:', answer);
```

## Rate Limiting

The public chat endpoint enforces per-visitor daily rate limits.

| Property | Details |
|----------|---------|
| **Mechanism** | Cookie-based visitor tracking (`visitor_id`) |
| **Default Limit** | Configurable via `public_chat_daily_limit` |
| **Scope** | Per visitor per day (resets at midnight) |
| **Storage** | In-memory (not persisted; resets on server restart) |
| **Response on Exceed** | HTTP `429` with descriptive error message |

:::note
Rate limits are tied to the `visitor_id` cookie, not the IP address. If a visitor clears their cookies, they may receive a new quota.
:::
